================================================================================
                       DATABASE SCHEMA
================================================================================

5 Tables with Full Indexes:

1. tenants (Organization container)
   - id (UUID, PK)
   - name, slug (unique), owner_id (FK User)
   - tier, schema_name (PostgreSQL schema)
   - max_users, max_api_keys, max_monthly_api_calls
   - settings (encrypted JSONB), features (JSONB)
   - is_active, is_paused, suspended_reason
   - created_at, updated_at

2. tenant_members (RBAC)
   - id (UUID, PK)
   - tenant_id (FK), user_id (FK)
   - role (owner/admin/member/viewer)
   - permissions (custom dict)
   - invited_by, invited_at, accepted_at
   - is_active

3. tenant_api_keys (Service integrations)
   - id (UUID, PK)
   - tenant_id (FK), key_hash (SHA256, unique)
   - key_prefix (visible), name, description
   - scopes (read/write/admin), ip_whitelist
   - rate_limit_per_minute
   - is_active, is_revoked, revoked_reason
   - last_rotated_at, must_rotate_by (90-day)
   - total_requests, total_errors
   - created_at, last_used_at

4. tenant_billing (Usage & costs)
   - id (UUID, PK)
   - tenant_id (FK, unique)
   - billing_period_start, billing_period_end
   - api_calls_used, storage_gb_used, compute_hours_used
   - base_tier_cost, overage_cost, total_cost
   - status (pending/paid/failed/refunded)
   - invoice_number, stripe_invoice_id
   - last_payment_attempt, payment_failed_reason

5. audit_logs (Compliance trail - immutable)
   - id (UUID, PK)
   - tenant_id (FK)
   - action, resource_type, resource_id, change_type
   - user_id (FK), user_email (denormalized)
   - ip_address, user_agent
   - status (success/failed/warning), reason
   - details (encrypted JSONB), old_values, new_values
   - created_at (indexed)

All tables have performance-optimized indexes.
